# ⚡ StructEditor

> Визуальный редактор содержимого страниц для MODX Revolution 2.8

**StructEditor** — это дополнение к административной панели MODX, которое заменяет стандартное поле ввода HTML-кода удобным интерфейсом с тремя режимами редактирования. Редактор не требует знания HTML — контент-менеджер работает с полями, кнопками и визуальным представлением страницы.

![StructEditor](https://raw.githubusercontent.com/commeta/StructEditor/refs/heads/main/img/full-screen.png "StructEditor")

---

## Преимущества StructEditor

**StructEditor** создан специально для MODX Revolution 2.8 и решает главную боль большинства сайтов: разросшиеся TV-поля и нестабильность обычных WYSIWYG-редакторов.

### 1. Одна страница — одно окно (полный отказ от большинства TV)
В стандартном MODX контент-менеджер вынужден заполнять 8–15 дополнительных полей (TV): «Заголовок секции», «Текст блока», «Изображение», «Ссылка», «HTML-виджет» и т.д.  

**StructEditor** переносит всё это в **единое поле «content»**:

- Автоматически разбирает HTML на логические секции (`<section>`, `<article>`, `<header>` и т.д.).
- Каждую секцию представляет как карточку с готовыми полями.
- Все виджеты, эмбеды, таблицы и сложные блоки остаются внутри одного документа.

**Результат для администратора:** можно удалить 70–90 % TV-полей из шаблона. Шаблон становится чище, страница редактирования загружается быстрее, а контент-менеджер не путается в десятках вкладок.

### 2. Безопасный парсинг сложных HTML-виджетов (где обычные WYSIWYG-редакторы ломаются)

Обычные JavaScript-WYSIWYG (TinyMCE, CKEditor, Redactor и др.):
- «пожирают» атрибуты;
- переписывают вложенные теги;
- удаляют `data-*`, `style`, `onload`, SVG и кастомные виджеты.

**StructEditor** работает иначе:

- **Токенизация защищённых блоков** (`<script>`, `<style>`, `<?php ?>`, `<!-- -->`, `<svg>`, `<math>`) — они полностью сохраняются и не попадают в редактируемые узлы.
- **DOMParser + walkMark** анализирует дерево до глубины 40 уровней, маркирует только редактируемые элементы по строгим правилам (TEXT_TAGS + ATTR_MAP + LEAF_CONTAINERS).
- Защищённые блоки заменяются плейсхолдерами `<div data-sev8ph="…">` и восстанавливаются при сериализации (detokenize).

**Результат:** любой сложный HTML-виджет (карусель на чистом HTML, форма с data-атрибутами, эмбед YouTube с параметрами, таблица с colspan и т.д.) остаётся нетронутым. Даже после десятков правок в визуальном режиме код остаётся валидным.

### 3. Три режима редактирования под разные задачи

| Режим          | Для кого                          | Когда использовать                              |
|----------------|-----------------------------------|-------------------------------------------------|
| **Структура**  | Контент-менеджеры                 | Основная работа: поля + дерево + форматирование |
| **Визуальный** | Быстрое редактирование заголовков | Inline-редактирование h1–h6 и клик по элементу  |
| **Исходник**   | Веб-мастера и разработчики        | Вставка готовых блоков, правка PHP-вставок      |

Переключение одним кликом. Все режимы синхронизированы в реальном времени.

### 4. История изменений + IndexedDB (50 шагов)
- Автоматическое сохранение каждого состояния в браузере (привязка к ID ресурса).
- Undo/Redo даже после перезагрузки страницы.
- Авто-инвалидация истории при внешнем изменении документа (защита от конфликтов).

### 5. Полная интеграция с MODX
- Автоматически подхватывает ACE-редактор или нативный `<textarea>`.
- Перехватывает событие сохранения формы (`beforeSave`, кнопки «Сохранить», Ctrl+S).
- Перед отправкой формы всегда выполняет `syncNow()` — данные никогда не теряются.

### Кому это даёт максимальную выгоду
- **Контент-менеджерам** — скорость работы ×3, минимум ошибок, всё в одном окне.
- **Администраторам** — чистые шаблоны, меньше TV, проще поддержка.
- **Веб-мастерам** — возможность безопасно использовать сложные HTML-виджеты без страха, что редактор их сломает.

**StructEditor** — это не просто замена textarea. Это переход от «мозаики из TV-полей» к единому, структурированному и безопасному редактору, который понимает настоящий HTML, а не только то, что может отрендерить TinyMCE.

## Установка

Редактор подключается через плагин MODX на событие `OnManagerPageBeforeRender`:

```php
<?php
if ($modx->event->name == 'OnManagerPageBeforeRender') {
    if (isset($_GET['a']) && $_GET['a'] == 'resource/update') {
        $modx->controller->addJavascript('/assets/struct-editor.js');
    }
}
```

Файл `struct-editor.js` необходимо разместить в директории `/assets/` на сервере. После этого редактор автоматически появляется на странице редактирования ресурса вместо стандартного поля ввода.

### Элементы верхней панели

| Элемент | Описание |
|---|---|
| **⚡ StructEditor** | Логотип. Декоративный элемент, нажатие не предусмотрено |
| **Структура** | Переключает в режим редактирования по полям |
| **Визуальный** | Переключает в режим кликабельного предпросмотра |
| **Исходник** | Открывает сырой HTML для прямого редактирования |

## Режим «Структура»

Режим по умолчанию. Редактор анализирует HTML-код страницы, находит все текстовые блоки и атрибуты, и представляет их в виде отдельных полей.

### Переключение вида: Поля / Дерево

В верхней части рабочей области находятся две кнопки:

```
┌──────────────────────────────────────────────────────────┐
│  [ Поля ]  [ Дерево ]          ✂ Очистить пробелы        │
├──────────────────────────────────────────────────────────┤
│  ...                                                     │
```

- **Поля** — контент сгруппирован по секциям страницы (`<section>`, `<article>`, `<header>` и т.д.). Внутри каждой секции отображаются поля для редактирования текста и атрибутов.
- **Дерево** — иерархическая структура всех HTML-элементов с возможностью разворачивать и сворачивать ветки.

```

│                                                          │

│  │ Наш ассортимент                                    │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  <p> › section                    ⎘  +  ↑  ↓  ✕          │
│  ┌────────────────────────────────────────────────────┐  │
│  │ Широкий выбор товаров для дома и офиса.            │  │
│  │                                                    │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │

│                                                          │
└──────────────────────────────────────────────────────────┘
```

**Заголовок карточки** показывает тег секции, её название (берётся из первого заголовка `<h1>`–`<h4>` внутри) и количество редактируемых элементов. Клик по заголовку сворачивает / разворачивает карточку.

**Каждый элемент** содержит:
- Тег элемента (`<p>`, `<h2>`, `<img>` и т.д.) и путь в дереве DOM
- Текстовое поле для редактирования содержимого
- Поля для атрибутов (для изображений — `src`, `alt`, `width`, `height`; для ссылок — `href`, `title`, `target` и т.д.)
- Для изображений — миниатюра превью, которая обновляется при изменении `src`
- Кнопки управления элементом

**Редактирование текста.** Нажмите на поле и начните вводить текст. Изменения сохраняются автоматически с небольшой задержкой. Поля поддерживают вставку HTML-тегов — например, можно вручную добавить `<strong>слово</strong>` или воспользоваться панелью форматирования.

### Вид «Дерево»

![StructEditor tree](https://raw.githubusercontent.com/commeta/StructEditor/refs/heads/main/img/struct-tree.png "StructEditor tree")


```
┌──────────────────────────────────────────────────────────────┐
│ ▶ <div>  .container  3 эл.           ⎘  ↑  ↓  ✕             │
│   ▶ <section>  .hero  2 эл.          ⎘  ↑  ↓  ✕             │
│   │  · <h1>  Добро пожаловать        ⎘  ↑  ↓  ✕             │
│   │    ┌────────────────────────┐                            │
```

Дерево показывает вложенность HTML-элементов с направляющими линиями. Цвет тега указывает на тип элемента:
- **Синий** — контейнер с дочерними элементами
- **Зелёный** — листовой элемент (содержит редактируемый текст или атрибуты)

Клик по контейнеру разворачивает или сворачивает его содержимое. Клик по листовому элементу открывает поле редактирования прямо под ним.

### Кнопка «✂ Очистить пробелы»

Проходит по всем текстовым полям и удаляет лишние пробелы, переносы строк и отступы в начале и конце значений. Полезно после вставки текста из Word или других редакторов.

### Работа в визуальном режиме

**Шаг 1.** Наведите курсор на элемент — он подсветится синей рамкой.

**Шаг 2.** Кликните на элемент.

**Для заголовков** (`<h1>`–`<h6>`) поле ввода появляется прямо на месте элемента — текст можно редактировать inline:

```
┌──────────────────────────────────────────────────────────────┐
│  ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐       │
│  │ Заголовок страницы█                               │       │
│  └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘       │
└──────────────────────────────────────────────────────────────┘
```

Нажмите **Enter** или кликните вне поля — изменение сохранится. Нажмите **Escape** — изменение будет отменено.

**Для остальных элементов** открывается всплывающее окно рядом с элементом:

```
        ┌─────────────────────────────────────────────┐
        │ ✏ <p>                                   [✕] │
        ├─────────────────────────────────────────────┤
        │ ┌─────────────────────────────────────────┐ │
        │ │ Текст абзаца можно редактировать здесь  │ │
        │ │                                         │ │
        │ └─────────────────────────────────────────┘ │
        │                                             │
        │                  [Отмена] [✓ Сохранить]     │
        └─────────────────────────────────────────────┘
```

Для элементов с атрибутами (изображения, ссылки) в окне дополнительно отображаются поля атрибутов.

> **Важно.** Ссылки в визуальном режиме заблокированы — переходы по ним не происходят, чтобы случайный клик не уводил со страницы редактирования.

## Режим «Исходник»

![StructEditor source](https://raw.githubusercontent.com/commeta/StructEditor/refs/heads/main/img/source.png "StructEditor source")


Прямой доступ к HTML-коду страницы. Подходит для опытных пользователей, которым нужно вставить готовый HTML-блок или исправить разметку вручную.

```
┌──────────────────────────────────────────────────────────────┐
│ ⚠ Исходный HTML. Ctrl+S / Ctrl+Enter — применить.            │
│                             [⚙ Форматировать]  [✓ Применить] │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  <section class="hero">                                      │
│    <h1>Заголовок страницы</h1>                               │
│    <p>Текст описания.</p>                                    │
│  </section>                                                  │
│  <section class="catalog">                                   │
│    ...                                                       │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│                               47 стр. / 1842 симв.           │
└──────────────────────────────────────────────────────────────┘
```

### Элементы режима «Исходник»

**Текстовая область** — поле для ввода HTML. Поддерживает отступы клавишей `Tab` (вставляет два пробела).

**Счётчик** в правом нижнем углу показывает количество строк и символов — обновляется в реальном времени.

**⚙ Форматировать** — автоматически расставляет отступы и переносы строк в HTML-коде для удобства чтения. Форматирование применяется немедленно.

**✓ Применить** — вручную применяет изменения из текстовой области к редактору. Также происходит автоматически при потере фокуса (клик вне текстовой области) и по горячим клавишам.

> **Примечание.** Редактор безопасно обрабатывает специальные блоки кода: теги `<script>`, `<style>`, PHP-вставки `<?php ... ?>`, комментарии и SVG-изображения сохраняются в неизменном виде и не отображаются как редактируемые элементы.

## Панель форматирования

Расположена под вкладками, активируется при клике в любое текстовое поле (в режимах «Структура» и «Исходник»):

```
ФОРМАТ:  B  I  U  S │ M  </>  "q"  +ins  x²  x₂  Aa │ A  ■  Размер ▾ │ Ссылка  ABR  .cls │ ­  &nbsp;  —  «»  <br> │ ✕ fmt
```

Пока ни одно поле не выбрано, панель отображается в затемнённом виде и недоступна.

### Инструменты форматирования

| Кнопка | Тег | Действие |
|---|---|---|
| **B** | `<strong>` | Жирный текст |
| _I_ | `<em>` | Курсив |
| U̲ | `<u>` | Подчёркнутый |
| ~~S~~ | `<s>` | Зачёркнутый |
| M | `<mark>` | Выделение маркером |
| `</>` | `<code>` | Форматирование кода |
| "q" | `<q>` | Инлайн-цитата |
| +ins | `<ins>` | Вставленный текст |
| x² | `<sup>` | Надстрочный индекс |
| x₂ | `<sub>` | Подстрочный индекс |
| Aa | `<small>` | Уменьшенный текст |

### Цвет и размер

**A** — цвет текста. Открывает системный выбор цвета браузера. Полоска под буквой показывает текущий выбранный цвет.

**■** — цвет фона. Аналогично, но добавляет фоновую заливку.

**Размер ▾** — выпадающий список с предустановленными размерами шрифта: от «Очень мелкого» (0.7em) до «Очень большого» (2em).

### Специальные элементы

**Ссылка** — открывает поле для ввода URL:

```
┌────────────────────────────────┐
│ https://...                    │
│ ☐ Новая вкладка                │
│           [Вставить ссылку]    │
└────────────────────────────────┘
```

Заполните адрес, при необходимости отметьте «Новая вкладка» (добавит `target="_blank"`), нажмите **Enter** или кнопку. Если перед открытием панели был выделен текст — он станет текстом ссылки.

**ABR** — аббревиатура `<abbr title="...">`. Введите расшифровку в поле и нажмите «Вставить abbr». Браузер будет показывать расшифровку при наведении курсора.

**.cls** — обернуть выделенный фрагмент в `<span class="...">`. Удобно для применения стилей CSS.

### Вставка символов

| Кнопка | Что вставляет |
|---|---|
| `­` | Мягкий перенос (`&shy;`) — браузер переносит слово здесь при нехватке места |
| `&nbsp;` | Неразрывный пробел — предотвращает перенос строки между словами |
| `—` | Длинное тире |
| `«»` | Кавычки-ёлочки. При выделенном тексте оборачивает его; без выделения вставляет пустую пару |
| `<br>` | Принудительный перенос строки |

**✕ fmt** — очищает форматирование из выделенного фрагмента. Удаляет теги `<strong>`, `<em>`, `<u>`, `<span>`, `<code>`, `<mark>` и другие инлайн-теги, оставляя только текст.

### Как применять форматирование

1. Кликните в поле редактирования — панель станет активной.
2. Выделите нужный текст мышью или клавиатурой.
3. Нажмите кнопку на панели форматирования.

Если ничего не выделено и кнопка предполагает обёртку — вставится шаблон вида `<strong>текст</strong>` с уже выделенным словом «текст» для замены.

## История изменений

Редактор автоматически сохраняет историю правок в браузере (IndexedDB). История привязана к конкретному ресурсу MODX и сохраняется между сессиями.

```
        Кнопки в верхней панели:
        ┌────────────┐  ┌────────────┐
        │ ↩ Отмена   │  │ ↪ Повтор   │
        └────────────┘  └────────────┘
          (серая = нет     (серая = нет
           предыдущих        следующих
           состояний)        состояний)
```

**Undo (↩ Отмена)** отменяет последнее сохранённое состояние. Максимальная глубина истории — 50 шагов.

**Redo (↪ Повтор)** возвращает отменённое состояние.

> **Примечание.** Если страница была изменена напрямую (например, через другой браузер или другой пользователь сохранил страницу), история автоматически сбрасывается, чтобы не перезаписать чужие правки.

## Операции с элементами

Рядом с каждым элементом в режимах «Структура» (вид «Поля») и «Дерево» расположены кнопки управления:

```
<p> › section.intro    ⎘  +  ↑  ↓  ✕
                        │  │  │  │  └── Удалить элемент
                        │  │  │  └───── Переместить вниз
                        │  │  └──────── Переместить вверх
                        │  └─────────── Добавить такой же элемент после
                        └────────────── Клонировать (скопировать)
```

> **Кнопка «+»** доступна только для элементов, которые логически повторяются: `<li>`, `<p>`, `<dt>`, `<dd>`, `<tr>`, `<option>`, `<blockquote>`.

### Клонирование (⎘)

Создаёт точную копию элемента и вставляет её сразу после оригинала. Копируется весь HTML-код элемента, включая вложенные теги и атрибуты.

### Добавление (+)

Создаёт новый пустой элемент того же типа и вставляет его после текущего. Для таблиц (`<tr>`) новая строка создаётся с теми же ячейками (`<td>`/`<th>`), но с пустым содержимым.

### Перемещение (↑ ↓)

Меняет позицию элемента относительно соседних элементов того же уровня.

### Удаление (✕)

Перед удалением показывается диалог подтверждения с тегом и первыми 60 символами текста элемента. Удаление нельзя отменить стандартной кнопкой «Отмена», однако можно воспользоваться историей (Undo).

## Сохранение

Редактор работает в связке со стандартными механизмами сохранения MODX. Изменения не теряются, если соблюдать следующий порядок.

### Как работает синхронизация

Когда редактируется текст в полях, изменения автоматически записываются во внутреннее состояние редактора с небольшой задержкой (около 0,6 секунды после последнего нажатия клавиши). Это отражается в строке состояния:

```
● изменено    →    Синхронизировано ✓
```

Чтобы изменения попали в базу данных MODX, необходимо нажать стандартную кнопку сохранения MODX («Сохранить» в панели управления). Редактор автоматически перехватывает это действие и синхронизирует данные перед отправкой формы.

### Способы сохранить страницу

1. **Кнопка «💾 Синхр.»** — только синхронизирует редактор с полем MODX, не сохраняет в базу данных.
2. **Ctrl + S** — синхронизирует и инициирует сохранение через кнопку MODX.
3. **Стандартная кнопка «Сохранить» MODX** — редактор автоматически синхронизируется перед отправкой.

> **Совет.** Перед закрытием вкладки убедитесь, что в строке состояния отображается «Синхронизировано ✓», а не «● изменено».

## Часто задаваемые вопросы

**Редактор не появился — вместо него стандартное поле MODX**

Проверьте: файл `struct-editor.js` находится по пути `/assets/struct-editor.js`; плагин активен и настроен на событие `OnManagerPageBeforeRender`; вы открыли страницу редактирования ресурса (URL содержит `?a=resource/update`).

---

**Часть содержимого страницы не отображается в режиме «Структура»**

Редактор показывает только те элементы, которые содержат текст или значимые атрибуты. Блоки `<script>`, `<style>`, PHP-вставки, комментарии и SVG намеренно скрыты — они доступны в режиме «Исходник».

---

**После редактирования изображение сломалось или потеряло пропорции**

Проверьте поля `width` и `height` в атрибутах изображения (режим «Структура», вид «Поля»). Если задана только ширина, высота рассчитается автоматически браузером — удалите ненужное значение или установите правильное.

---

**Кнопка «↩ Отмена» неактивна**

Это означает, что в истории нет предыдущих состояний. История начинает накапливаться с момента первого изменения. При открытии страницы история восстанавливается из браузера — если вы впервые открываете этот ресурс, история пуста.

---

**Нужно вернуться к стандартному редактору MODX**

Нажмите кнопку **✎ Textarea** (или **↩ ACE**, если используется ACE-редактор) в правой части верхней панели. Редактор скроется, стандартное поле снова станет видимым. Все изменения предварительно синхронизируются.

---

**После вставки текста из Word появились лишние пробелы**

Воспользуйтесь кнопкой **«✂ Очистить пробелы»** в режиме «Структура» (вид «Поля»). Функция пройдёт по всем полям и уберёт лишние пробелы, переносы и отступы.

---

**Всплывающее окно редактирования открывается в неудобном месте**

Окно автоматически позиционируется рядом с элементом и подстраивается под края экрана. Если элемент находится у края, окно сдвигается внутрь видимой области. Изменить расположение вручную нельзя.

---

**Что делает кнопка «⊞ Весь экран»?**

Разворачивает панель редактора на весь экран браузера, перекрывая интерфейс MODX. Удобно при работе с большим объёмом текста. Повторное нажатие (надпись меняется на «⊞ Свернуть») возвращает редактор в исходный размер.

## Системные переменные

### Основные константы (`CFG`)

Все параметры парсинга, поведения и хранения данных сосредоточены здесь.

| Ключ                | Значение по умолчанию          | Описание |
|---------------------|--------------------------------|----------|
| `TA_SELECTORS`      | массив селекторов              | Список CSS-селекторов для автоматического поиска нативного `<textarea>` MODX. Первый найденный элемент используется как источник данных. |
| `POLL_MS`           | `250`                          | Интервал опроса (мс) при старте редактора. |
| `POLL_MAX`          | `160`                          | Максимальное количество попыток поиска редактора перед таймаутом (≈40 секунд). |
| `MARK`              | `'data-sev8'`                  | Атрибут, которым помечаются редактируемые узлы DOM (внутренний). |
| `PH_ATTR`           | `'data-sev8ph'`                | Атрибут плейсхолдеров для защищённых блоков (`<script>`, `<style>` и т.д.). |
| `TEXT_TAGS`         | объект с тегами                | Теги, считающиеся текстовыми блоками (`h1`–`h6`, `p`, `li`, `td` и др.). Если внутри нет других текстовых тегов — элемент становится редактируемым. |
| `ATTR_MAP`          | объект сопоставления           | Для каких тегов редактируются атрибуты (`a`: `href/title/target`, `img`: `src/alt/width/height/title` и т.д.). |
| `LEAF_CONTAINERS`   | `{ div:1, span:1 }`            | Контейнеры, которые могут стать «листовыми» элементами, если содержат минимум текста. |
| `LEAF_MIN_CH`       | `2`                            | Минимальное количество символов в текстовом содержимом, при котором `div`/`span` становится редактируемым. |
| `SKIP`              | объект исключений              | Теги, которые никогда не маркируются и не редактируются (`script`, `style`, `svg`, `br`, `input` и др.). |
| `MAX_DEPTH`         | `40`                           | Максимальная глубина рекурсивного обхода DOM (защита от бесконечных вложений). |
| `HISTORY_DB`        | `'sev8history'`                | Название базы IndexedDB для хранения истории. |
| `HISTORY_STORE`     | `'snapshots'`                  | Название object store внутри базы. |
| `HISTORY_MAX`       | `50`                           | Максимальное количество сохранённых состояний (Undo/Redo). |

### Дополнительные внутренние настройки

- **TOKEN_PATTERNS** — массив регулярных выражений (9 шт.), определяющих защищённые блоки, которые **никогда** не попадают в режим редактирования.  
  Пример: `<!--[\s\S]*?-->`, `<script[\s\S]*?</script>`, `<\?php[\s\S]*?\?>` и т.д.  
  Эти блоки заменяются плейсхолдерами и полностью восстанавливаются при сохранении.

- **TREE_CONTAINERS** и **TREE_SKIP** — используются только в режиме «Дерево» для построения иерархии.

### Как изменить настройки

1. Откройте файл `/assets/struct-editor.js`.
2. Найдите блок `/* ════════ КОНФИГ ════════ */` (строка ~80) или `/* ════════ ЦВЕТА ════════ */`.
3. Измените нужные значения.
4. Сохраните файл.
5. Перезагрузите страницу редактирования ресурса (Ctrl + F5).

**Рекомендации по кастомизации:**
- Для брендирования измените только `C` (все 30 цветов).
- Для увеличения производительности на очень больших страницах уменьшите `MAX_DEPTH` до 25–30.
- Для большего количества откатов увеличьте `HISTORY_MAX` до 100 (память браузера).
- Если нужно добавить поддержку новых атрибутов — расширьте `ATTR_MAP` (например, `video: ['src', 'poster', 'controls']`).

Все изменения сохраняют полную совместимость с MODX Revolution 2.8 и не требуют перекомпиляции или дополнительных модулей.

### Датасеты

Датасет — это HTML-содержимое, которое загружается в поле `content` эмулируемого ресурса MODX. Редактор парсит его и монтируется, как при обычном открытии страницы в административной панели. При выборе датасета из списка редактор перезагружается автоматически; кнопка **↻ Load** перезагружает текущий датасет принудительно.

| № | Название | Что проверяет |
|---|---|---|
| 1 | Заголовки + абзацы | Базовый парсинг `h1`–`h6`, `p`, `strong`, `em` |
| 2 | Полный semantic HTML5 | `article`, `section`, `header`, `footer`, `blockquote`, `time`, `details` |
| 3 | Ссылки + медиа | `a`, `img`, `video`, `audio`, `iframe`, `figure` |
| 4 | Таблицы + списки + форма | `table`, `ul`, `ol`, `dl`, `form` |
| 5 | Вложенные секции | Многоуровневая структура `section` внутри `section` |
| 6 | Токены (script/style/php) | Защита `<script>`, `<style>`, PHP-тегов, комментариев, `<noscript>` |
| 7 | Edge cases (unicode/XSS) | Эмодзи, японский, арабский, попытки XSS-инъекций |
| 8 | Смешанный — все типы | Комбинация всех предыдущих типов содержимого |

Рекомендуется запускать **Run All** на каждом датасете по очереди, особенно при проверке датасета 6 (токены) и датасета 7 (edge cases) — они наиболее показательны для выявления регрессий после изменений в `struct-editor.js`.

### Группы тестов (suite-группы)

Тесты разделены на 25 групп, каждая проверяет отдельный аспект работы редактора.

#### Юнит-тесты (изолированные функции)

| Группа | Что проверяется |
|---|---|
| **Unit: simpleHash** | Хэш-функция: возвращает строку, детерминирована, чувствительна к регистру и пробелам, устойчива к коллизиям (500 случайных строк — менее 5 совпадений), работает с unicode и длинными строками. |
| **Unit: tokenize/detokenize** | Защита `<script>`, `<style>`, `<svg>`, PHP-тегов, HTML-комментариев, `<noscript>`, `<template>`, `<meta>` — они заменяются плейсхолдерами и точно восстанавливаются. Проверяется 50 последовательных циклов токенизации на стабильность, отсутствие плейсхолдеров в финальном выводе, корректность при токене в начале и в конце строки. |
| **Unit: walkMark** | Алгоритм обхода DOM: какие теги помечаются как редактируемые (`h1`–`h6`, `p`, `li`, `td`, `img`, `a` и др.), какие пропускаются (`script`, `input`, `br`, `hr`), корректность определения типа (`text` / `attrs` / `both`), поведение `div` и `span` как leaf-container, ограничение глубины `MAX_DEPTH`. |
| **Unit: serialize** | Сборка итогового HTML из помеченного дерева: отсутствие служебных атрибутов `data-sev8` и `data-sev8ph` в выходном коде, сохранение текста и атрибутов, идемпотентность повторной сериализации, корректность HTML-сущностей, стабильность 100 последовательных вызовов. |
| **Unit: fmtHTML** | Форматирование (расстановка отступов): блочные элементы переносятся на новые строки, содержимое `<pre>` не трогается, атрибуты сохраняются, unicode-символы не искажаются, токены пережив форматирование не теряются, `S.tokens` восстанавливается после вызова (без утечки в следующий `tokenize`), результат остаётся валидным HTML. |
| **Unit: History internals** | Внутренний стек истории: `histPush` добавляет запись только при изменении контента, дедуплицирует идентичные вызовы, обрезает «будущее» при ветвлении, не превышает `HISTORY_MAX`; `histUndo` и `histRedo` корректно двигают указатель и не падают на границах стека; флаг `histBusy` блокирует запись. |

#### Интеграционные тесты (редактор в браузере)

| Группа | Что проверяется |
|---|---|
| **Integration: Mount** | Редактор смонтировался: элементы `#sev8`, `#sev8-tb`, ровно три вкладки с нужными подписями, кнопки Undo/Redo/Sync, форматная строка `#sev8-fmtrow`, рабочая область `#sev8-area`, строка статуса `#sev8-st`, нативная `<textarea>` скрыта (`display:none`), CSS-блок `#sev8-css` подключён, корневой элемент имеет `display:flex`, кнопка Undo отключена при пустой истории. |

| **Integration: cleanSpaces** | Функция `cleanSpaces()`: убирает ведущие и хвостовые пробелы из поля, схлопывает несколько пробелов в один, после вызова вызывает `syncNow` и обновляет нативный `<textarea>`. |
| **Integration: Source Tab** | Вкладка «Исходник»: `#sev8-src-ta` с HTML-кодом присутствует, кнопки «Формат» и «Применить» есть, Tab вставляет ровно 2 пробела, Ctrl+Enter применяет изменения в нативный `<textarea>`, blur также применяет изменения, «Формат» перефоматирует и результат парсится без ошибок, счётчик символов и строк обновляется при вводе. |

| **Integration: Debounce** | Тайминги дебаунса: после события `input` нативный `<textarea>` **не** обновляется в течение 300 мс (дебаунс активен); событие `change` обновляет немедленно (`syncNow`); после ~1 секунды дебаунс завершается и нативный `<textarea>` получает обновлённое значение. |
| **Integration: Edit & Attrs** | Изменение поля в режиме «Структура» отражается в нативной `<textarea>` после синхронизации; при `target="_blank"` автоматически проставляется `rel="noopener"`; поле `alt` у изображений доступно; изменение поля `src` у `<img>` мгновенно обновляет превью. |
| **Integration: Struct Ops** | Кнопки управления элементами: клонирование добавляет новый элемент с тем же `data-id`-префиксом; перемещение вверх/вниз физически меняет порядок узлов в DOM; отмена удаления не меняет количество строк; подтверждённое удаление уменьшает количество строк на 1; кнопка «Добавить» вставляет пустой элемент (только для `li`, `p`, `tr` и аналогичных тегов). |
| **Integration: Format Bar** | Панель форматирования: существует и содержит 5+ кнопок; затухает без фокуса; активируется при фокусе на поле структуры; кнопки оборачивают выделенный текст в соответствующие теги: `<strong>`, `<em>`, `<u>`, `<s>`, `<mark>`, `<code>`, `<q>`, `<ins>`, `<sup>`, `<sub>`, `<small>`; кнопки вставки добавляют `&nbsp;`, `—`, `<br>`; «Очистить формат» убирает теги из выделения; при отсутствии выделения вставляется шаблон-заготовка. |
| **Integration: Fullscreen** | Кнопка «⊞» переводит редактор в `position:fixed`, добавляет `overflow:hidden` на `<body>`; повторное нажатие снимает оба свойства и возвращает к исходному состоянию. |
| **Integration: Tab Switching** | В каждый момент активна ровно одна вкладка; после цикла «Структура → Исходник → Структура» нативное поле не пустеет; цикл «Структура → Визуальный → Структура» удаляет `#sev8-vis` и восстанавливает область; 5 быстрых переключений подряд не ломают редактор; содержимое `<textarea>` в Source Tab совпадает с результатом `serialize()`. |

### Уровни логирования

Тест-стенд пишет диагностику в консоль браузера (F12 → Console). Уровень выбирается в выпадающем списке **Log** в верхней панели:

| Уровень | Что попадает в консоль |
|---|---|
| `info` | Ключевые события: инициализация, старт/финиш прогона, ошибки монтирования |
| `debug` | Дополнительно: каждый шаг монтирования, загрузка датасетов, результаты отдельных тестов |
| `trace` | Максимальная детализация: все внутренние вызовы (полезно при отладке конкретного упавшего теста) |

Сообщения имеют формат `[TEST:LEVEL][TAG] текст` — их удобно фильтровать по тегу в консоли (например, `RUNNER`, `SETUP`, `TESTSTAND`).

### Типовые сценарии использования

**Проверка после правки `struct-editor.js`**

1. Загрузите `struct-editor.html` в браузере.
2. Убедитесь, что индикатор показывает *✓ mounted*.
3. Нажмите **▶ Run All** — дождитесь завершения.
4. Если все тесты зелёные — правки безопасны. Если есть красные — прочитайте сообщение об ошибке и откатите или исправьте изменение.
5. Повторите для датасетов 6 и 7, если правки касались токенизации или парсинга сложных блоков.

**Локализация проблемы**

1. Выберите датасет, на котором воспроизводится проблема.
2. Нажмите **▶ Run All**, найдите красную группу.
3. Кликните на кнопку этой группы в фильтр-баре.
4. Нажмите **▶ Suite** — запустятся только тесты этой группы.
5. Переключите **Log** в `trace` и изучите консоль.
6. Нажмите **↺ Failed** после исправления — перегонит только упавшие тесты без полного прогона.

**Проверка перед сдачей проекта**

Последовательно выберите все 8 датасетов и на каждом выполните **▶ Run All**. Результатом должен быть прогресс-бар без красных делений и отсутствие ✗ в группах **Data Integrity**, **Token Integrity** и **Performance**.
